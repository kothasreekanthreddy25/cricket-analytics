7. Implementation Roadmap
Phase 1 — Setup (Week 1)
Set up Google Cloud project, enable YouTube Data API v3, create OAuth credentials. Set up Hetzner VPS, install Node.js + FFmpeg. Set up YouTube channel for live streaming. Test RTMP connection manually.
Phase 2 — Core Streaming (Week 2)
Build Node.js streaming service: Roanuz ball-by-ball → GPT-4o commentary → Google TTS audio. FFmpeg renders scorecard overlay + audio as H.264 video stream to YouTube RTMP. Test with a real live match.
Phase 3 — Website Integration (Week 3)
Embed YouTube player on /live/[matchKey] page. Auto-detect when a stream goes live and show embed. Link match page to YouTube stream. Add "LIVE ON YOUTUBE" badge on match cards.
Phase 4 — Voice Upgrade (After monetization)
Upgrade from Google TTS to ElevenLabs for premium voice quality. Add Hindi / Tamil commentary track option. Add multi-language support for wider audience reach.
Phase 5 — Automation (Ongoing)
Auto-start stream when Roanuz detects a match going live. Auto-stop stream when match ends. Auto-generate stream title, description, thumbnail using GPT-4o. Schedule streams 30 mins before match start.

📈 Monetization Break-Even
YouTube pays roughly $1–4 per 1,000 views (RPM) for cricket content. At 30 matches/month streamed, with even 5,000 views per stream, that's 150,000 views × $2 RPM = $300/month in ad revenue — well covering the $79/month infrastructure cost. Break-even requires approximately 1,000 subscribers + 4,000 watch hours to unlock monetization.
